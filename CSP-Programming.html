Part 1 - Getting Started

Creating a GitHub Account

Git is a distributed version control system (VCS) for tracking changes to projects, a tool to manage your source code history. GitHub is a hosting service for Git repositories, a service for projects that use Git.

1. Go to GitHub.com and sign up for an account.

    • I don't care what email you use
    • I don't care what your username is, but these are public so keep them appropriate.
    • If it asks, choose the free account
    • When it asks you questions about yourself, that is optional

2. Check your email and verify your your account.

Creating a Repl.it Account

We'll be using Repl.it as our in-browser IDE, so we don't need to set up personal computers and get to enjoy the auto-saving features.

1. Go to Repl.it and sign up for an account using your GitHub credentials (middle icon between Google and Facebook).

    • This is very important! If your accounts aren't linked - things won't work!

2. Grant permission for Repl.it to access your GitHub account

    • When it asks you questions about yourself, that is optional

GitHub is the primary point of access - it's where you'll be able to access the Repl.it editor (via a button link in the added to the Readme) and see if your assignment has passed all the tests, but more on that later. Troubleshooting

Rep.it button isn't showing up

  1. Sign into your Repl.it account using the GitHub login button (you may need to use the Sign Up button workaround talked about in an earlier announcement)
  2. Click on the blue plus sign in the upper right corner to create a new repl.
  3. Click on the Import from GitHub Tab and select your repo.
  4. That's it. This repo is linked and will allow you to push your code to GitHub.

I can't sign back into Repl.it/I can't access my repo on Repl.it

1. Close the window and try again with the "Sign Up" button using your GitHub account (must be signed into GitHub first).
